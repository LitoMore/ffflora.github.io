How to Run Flask by Command

After installing/updating the flask package with $conda$, one could use the following command to run the scripts:

$ flask run 
# This command runs the app.py or wsgi.py under the current dir automatically.

And user could find the server running on $http://localhost:5000$, the port by default is 5000.

If you are going to run the scripts other than the two filenames mentioned above, you need to specify the environment variable FLASK_APP by:

$ set FLASK_APP = hello

But this is not a smart way to manage all the env-variables.

A better way of doing that is to use the python-dotenv package, then when using flask run or other commands, it will automatically load the environment variables from .flaskenv and .env files.

$ pipenv install python-dotenv
# this would be installed under the project folder
$ pipenv shell 
# this command used to activate the virtualenv.

The priority of Flask to load the environment variables is:

hand set > .env > .flaskenv

.env and .flaskenv files needed to be created under the project’s root address.

.env is used to contain the sensitive environment variables, like username and passwords, so this file cannot be upload the GitHub.

.flaskenv contains information all the public env-variables that related to the flask, such as FLASK_APP.

Set the Production Environment

When we are in the development, we could set the production environment to development, which would enable all the development features.

Now add the FLASK_ENV to .flaskenv file:

FLASK_ENV=development

or just type the following in the terminal:

$ set FLASK_ENV = development

And this command will activate the debug mode, which triggers the Werksrug Debugger.

Python Shell

run the commandflask shell to start flask shell, and could exit by the commands exit() or quit() or Ctrl+Z+Enter.

Config

Mostly, user can access the api app.config to setup and acquire the config settings, and it usually is a dictionary file.

There are many ways to load the settings:

app.config['ADMIN_NAME']='Flora'
# tips: the settings must in capital letters, otherwise they won't be read.
app.config.update(
	TESTING = True,
	SECRET_KEY = '....')

Or config info could also be saved in JSON or python scripts format.

URL and Endpoint

Flask provides url_for() function to get the urls. When the URL rules that defined in the routes are modified, the function would always return the correct URL.

The first param for url_for() is the endpoint, which is used to mark the view function and its corresponding URL rules. The default value is the name of the view function.

@app.route('/')
def index():
	return 'Hello Flora!'

The view function is index, so when we do url_for('index') here, we would get the URL: '/'.

Flask Command

We could also self define some flask command:

declare a function + add app.cli.command() => then we get a flask command.

@app.cli.command()
def hello():
	click.echo('Hello, human!')

and this command could be run by:

$ flask hello
